Project Overview

DataHearth is a fully self-hosted home infrastructure built on a single Proxmox hypervisor. It replaces cloud services across media, AI, home automation, networking, and development — keeping all data and compute on-premises. Six virtual machines divide the workload cleanly by function, each with dedicated CPU and RAM allocations and PCIe passthrough where required.

Deployment Guide

Five Python generator scripts produce all Docker Compose files, setup scripts, config files, and dashboards. Each script is interactive — run it on your local machine, answer the prompts, then SCP the output folder to the target VM and run setup.sh.

Script Target VM Output Folder Generates
generate_sentinel.py Sentinel — 10.50.0.10 sentinel/ docker-compose.yml, setup.sh, servers.json, dashboard HTML, nginx.conf, DEPLOY.md
generate_compose.py Lexicon — 10.50.0.13 lexicon-deploy/ docker-compose.yml, setup.sh, dashboard HTML, DEPLOY.md, eXoDOS Flask app
generate_deepthought.py Deepthought — 10.50.0.14 deepthought/ docker-compose.yml, setup.sh, DEPLOY.md
generate_echo.py Echo — 10.50.0.20 echo/ docker-compose.yml, setup.sh, DEPLOY.md
01

Prerequisites — all generators

All generators require Python 3.8+. No pip packages needed — they use only the standard library. Run from a machine that can SSH/SCP to each VM on the 10.50.0.x network.

python3 --version   # must be 3.8+
02

Sentinel — Master Dashboard & Network Services

Run the generator, enable Omada and BorgBackup in the menu, add all other VMs in the linked servers screen. The default server list is pre-populated with Lexicon, Deepthought, and Echo.

python3 generate_sentinel.py
# ── Menu choices ──────────────────────────────────────────
# 1. IP: 10.50.0.10
# 2. Services: Omada=on
# 3. Host packages: BorgBackup=on, OpenVPN=on
# 5. Review/edit linked server list
# 7. Generate

# Deploy
scp -r sentinel/ user@10.50.0.10:~/
ssh user@10.50.0.10 'cd sentinel && sudo bash setup.sh'
# REBOOT if prompted, then:
ssh user@10.50.0.10 'sudo cp -r ~/sentinel/dashboard/* /mnt/disk0/sentinel/homepage/'
ssh user@10.50.0.10 'cd ~/sentinel && docker compose up -d'
Access the master dashboard at http://10.50.0.10 — status dots ping each server live.

Lexicon — Media & Library Services

The most complex generator. Use the multi-instance menus to add named Jellyfin instances. Enable desired singletons (Headway, Nextcloud, Kiwix, etc.). The RAM calculator shown in the summary screen helps you verify the 28 GB allocation is sufficient.

python3 generate_compose.py
# ── Key menu choices ──────────────────────────────────────
# 1. IP: 10.50.0.13
# 2. Configure Jellyfin instances (name each one)
# 3. Toggle singletons — enable Headway (port 8720)
# 4. Review RAM summary before generating
# 5. Generate

scp -r lexicon-deploy/ user@10.50.0.13:~/
ssh user@10.50.0.13 'cd lexicon-deploy && sudo bash setup.sh'
ssh user@10.50.0.13 'cd ~/lexicon-deploy && docker compose up -d'
Headway first-run: Before starting the headway container, run the one-time data build to download and index your chosen map region. This can take 1–4 hours for a large region and requires ~60 GB of disk space:

docker run --rm -v /mnt/disk0/lexicon/headway/data:/data headwaymaps/headway build --area north-america

Deepthought — Dual P40 LLM Server

Generates Ollama + OpenWebUI with SCHED_SPREAD for dual-GPU layer distribution. KV cache is set to q8_0 by default to halve VRAM cost on long contexts, leaving room for 70B quantised models on the 48 GB combined VRAM.

python3 generate_deepthought.py
# ── Menu choices ──────────────────────────────────────────
# 1. IP: 10.50.0.14
# 3. GPU count: 2 (both P40s)
# 4. Models dir: /mnt/disk0/Deepthought/models
# 5. KV cache: q8_0, keep_alive: -1
# 7. Generate

scp -r deepthought/ user@10.50.0.14:~/
ssh user@10.50.0.14 'cd deepthought && sudo bash setup.sh'
# REBOOT required after NVIDIA driver DKMS install
ssh user@10.50.0.14 'cd ~/deepthought && docker compose up -d'
# Pull a model to test both GPUs are used:
docker exec ollama ollama pull deepseek-r1:70b-distill-q4_K_M
docker exec ollama nvidia-smi  # both P40s should show VRAM usage
The setup.sh installs nvidia-kernel-dkms (proprietary 550.x) from Debian 13 non-free — not the open-kernel flavor, which doesn't support Pascal. A reboot is required after DKMS builds the kernel module.

Echo — STT + Small Reasoning Model

Single P40 (24 GB VRAM). faster-whisper large-v3 INT8 takes ~3.1 GB, leaving ~21 GB for Ollama. Qwen3 30B-A3B Q4_K_M at ~17 GB fits comfortably with ~2 GB to spare — the ideal always-on STT + reasoning setup.

python3 generate_echo.py
# ── Menu choices ──────────────────────────────────────────
# 1. IP: 10.50.0.20
# 3. GPU: NVIDIA, 1 GPU
# 4. Ollama: models dir /mnt/disk0/echo/ollama/models
# 5. Whisper: large-v3, INT8, GPU 0
# 7. Generate

scp -r echo/ user@10.50.0.20:~/
ssh user@10.50.0.20 'cd echo && sudo bash setup.sh'
# REBOOT required after NVIDIA driver DKMS install
ssh user@10.50.0.20 'cd ~/echo && docker compose up -d'
docker exec ollama ollama pull qwen3:30b-a3b-instruct-q4_K_M
Same Pascal driver rules as Deepthought: nvidia-kernel-dkms 550.x from Debian non-free. R580 is the final NVIDIA driver branch that supports GTX 10xx / Tesla P40 (Pascal). 590+ drops Pascal entirely.

NVIDIA Driver Notes

All three P40s (Deepthought ×2, Echo ×1) are Pascal architecture (sm_61). NVIDIA confirmed R580 is the final driver branch for Pascal. The 590+ series drops GTX 10xx and Tesla P4x entirely. The setup scripts install the correct 550.x series from Debian 13 non-free.
ItemCorrectIncorrect / Why it fails
Driver package nvidia-kernel-dkms nvidia-driver nvidia-open-kernel-dkms — Pascal not supported by open flavor
Driver series 550.x (from Debian 13 non-free) 590+ drops Pascal; 470.x is too old for CUDA 12
CUDA toolkit apt install nvidia-cuda-toolkit (Debian native) NVIDIA CUDA repo doesn't support Debian 13/Trixie yet
Kernel headers Install linux-headers-$(uname -r) FIRST Debian 13 / kernel 6.12 doesn't ship headers by default; DKMS silently fails
Flash Attention Leave unset — Pascal doesn't support it OLLAMA_FLASH_ATTENTION=1 silently falls back; no gain, possible confusion
Container Toolkit Official NVIDIA APT repo → nvidia-ctk runtime configure Must be installed AFTER Docker CE; must run nvidia-ctk to wire runtime
