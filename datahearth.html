DataHearth

A custom-built server designed to replace commonly used internet services with locally-run alternatives. Keep your data local, stay operational during disasters, and maintain full privacy control.

Proxmox Dual Xeon Docker Compose 6× PCIe x16 3× P40 + V100
01

Project Overview

🛡️ Disaster Preparedness

Functions without internet during natural or man-made disasters. Designed for prepping situations where backup power may be the only option.

⚡ Backup Power Compatible

Built to work with backup power systems, ensuring critical services remain operational when the grid goes down.

🔒 Privacy Control

Keep your data local instead of sending it to external servers. No more reliance on someone else's infrastructure.

Hardware

6× PCIe x16 slots (three are x8 electrical but accept x16 cards)
CPU
Dual Xeon — Lots of cores needed for all services
GPU Quantity Purpose
NVIDIA P40 3 LLM inference (lacks tensor cores for image/video generation)
NVIDIA V100 1 Image and video generation (has tensor cores)
💡 Note: Two P40s and the V100 are optional. Removing them significantly reduces GPU costs and eliminates the need for dual power supplies and the sync adapter.

Cooling Solution

Server-class GPUs lack built-in cooling fans. The solution uses 3D-printed shrouds with Arctic CPU fans. Blower fans were too loud. Single Arctic fans provided insufficient cooling, but two Arctic fans back-to-back increases pressure enough to cool cards quietly without sounding like a jet engine.

Build Configuration

An open-frame build was necessary due to:

  • Two-slot-wide GPU cards cannot all install directly on the motherboard
  • Cards with cooling are too large for standard cases
  • eATX motherboard doesn't fit standard cases
  • Dual power supply requirement
03

Software Stack

The main server runs Proxmox as the virtualization platform. Proxmox then runs several VMs, each with a Python helper script that lets you configure desired services and IP address, then generates a docker-compose.yml and installation script.

📝 Proxmox Tip: Instructions for running as open-source software and eliminating the subscription nag screen are included in proxmox/no-nag-patch.md.
