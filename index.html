An Internet at Home Project

DataHearth

A comprehensive homelab setup providing privacy-focused alternatives to cloud services. Built on Proxmox with multiple specialized VMs for media, automation, AI, and infrastructure management.

Hardware Overview

System Specifications
Enterprise-grade homelab server with dual Xeon processors and professional GPUs
Motherboard
X99 F8D Plus (dual socket, enterprise workstation)
Processors
Dual Intel Xeon (56 vCPU cores total)
Memory
92GB DDR4 (ECC registered memory)
Graphics Cards
3× Tesla P40 + 1× Tesla V100 + 1× GT 720
Storage
2TB NVMe + 4× 20TB SATA + LTO-5 SAS (80TB+ total)
Power Supply
Dual PSU (1200W + 850W) - open-frame configuration

An enterprise open-frame build configuration provides:

  • Multiple Tesla GPUs for serious AI/ML workloads (24GB VRAM per P40, 16GB HBM2 on V100)
  • ECC memory for data integrity in long-running processes
  • Massive storage capacity with LTO-5 tape backup for archival
  • Enterprise-grade reliability with dual Xeon architecture
  • Dual PSU for redundancy and high power delivery

03

Software Stack

The main server runs Proxmox as the virtualization platform. Proxmox then runs several VMs, each with a Python helper script that lets you configure desired services and IP address, then generates a docker-compose.yml and installation script.

📝 Proxmox Tip: Instructions for running as open-source software and eliminating the subscription nag screen are included in proxmox/no-nag-patch.md.

Project Structure

DataHearth/
├── Citadel/          # Home Assistant configuration & automations
├── deepthought/      # LLM server setup & model management  
├── echo/             # Voice assistant AI & Whisper config
├── lexicon/          # Media server configuration & scripts
├── proxmox/          # Hypervisor documentation & patches
├── Sentinel/         # Gateway services & network config
└── readme.md         # Project documentation & setup guide

Each directory contains Docker Compose configurations, setup scripts, and detailed documentation for deploying the respective services. The project is designed for modularity — you can deploy individual VMs or services based on your specific needs and hardware constraints.
